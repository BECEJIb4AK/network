network
=======
Первая задача:
tcp реализован частично, неоптимально. Основная функциональность - гарантированный обмен данными, если пакеты мешаются, теряются, портятся - есть.
Тест проверяет доставку, при том что пакеты случайно портятся/теряются. Может падать при старте, но если заработает, то отработает до конца.)

Вторая задача:
Локальный порт по умолчанию 1234. Логи - response_log и request_logю При запуске test.sh он он не получает response, просто не успевает, 
так как echo добавляет к строке завершение ввода. Если вместо test.sh запустить nc localhost 1234 и руками ввести HTTP-запрос, всё отработает идеально.
